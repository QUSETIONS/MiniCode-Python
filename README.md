## рҹҸ—пёҸ Architecture Deep Dive

### State Management (Zustand-Style)

```python
from minicode.state import create_app_store, set_busy, set_idle

# Create store
app_state = create_app_store({"model": "claude-sonnet-4"})

# Subscribe to changes
def on_change(new_state, old_state):
    print(f"State changed: {old_state.is_busy} вҶ’ {new_state.is_busy}")

app_state.subscribe(on_change)

# Update state
app_state.set_state(set_busy("read_file"))
# вҶ’ "State changed: False вҶ’ True"
```

### Engineering Governance (Built-in)

Every code generation automatically follows:

```
Iron Laws (8):
1. Theory first          5. Audit loop
2. Requirements first    6. Single sink (business/src/ = 1)
3. 1:1 binding           7. One-way dependencies (zero cycles)
4. Design-driven         8. No skipping phases

Package Structure:
my_package/
в”ңв”Җв”Җ port/
в”Ӯ   в”ңв”Җв”Җ port_entry/      # Entry points (can import anything)
в”Ӯ   в””в”Җв”Җ port_exit/       # Exit points (export interface)
в”ңв”Җв”Җ wrap/
в”Ӯ   в”ңв”Җв”Җ src/             # External library adapters
в”Ӯ   в””в”Җв”Җ config/          # Adapter configuration (zero deps)
в”ңв”Җв”Җ business/
в”Ӯ   в”ңв”Җв”Җ src/             # Business logic (CORE - exactly 1 sink)
в”Ӯ   в””в”Җв”Җ config/          # Business configuration (zero deps)
в”ңв”Җв”Җ test/
в”Ӯ   в”ңв”Җв”Җ src/             # Test code
в”Ӯ   в””в”Җв”Җ config/          # Test configuration (zero deps)
в””в”Җв”Җ docs/
    в”ңв”Җв”Җ requirements/    # User scenarios (pure, no implementation)
    в”ңв”Җв”Җ knowledge/       # Business rules & constraints (1:1 with requirements)
    в””в”Җв”Җ design/          # Technical design (maps to code structure)

Dependency Flow:
vendor/ вҶ’ port_entry вҶ’ wrap/src вҶ’ business/src вҶ’ port_exit
              вҶ‘                      вҶ‘
         (external libs)      (business config last)

Audit Checklist (Auto-Executed):
вң“ Audit 0: Knowledge вҶ” Requirements 1:1
вң“ Audit 1: Design вҶҗ Requirements + Knowledge coverage
вң“ Audit 2: Code вҶҗ Design isomorphism + Dependency compliance
вң“ Audit 3: business/src/ single sink + Package DAG
```

Use the `governance_audit` tool to check your code:

```python
# AI automatically runs this after code changes
governance_audit(action="full", path="my_package")

# Output:
# Governance Audit Result
# ==================================================
# вң“ PASSED - All governance rules satisfied
# 
# Dependencies: 12 edges
# Sink files:
#   business_src: 1 sink
#     - src/service.py
```
